推介话术参考
推介标准成功服务
开场引入
"您续订金蝶产品后，默认就包含了标准成功服务。这不只是'售后'，金蝶把它定义为帮您用好系统、防范风险的基础保障。主要包括金蝶云社区7×24的知识自助、智能客服随时解答，以及公有云环境全天候的巡检运维。遇到真正的致命故障——系统大面积无法使用——热线也可以第一时间响应。"
价值定位
"简单说，标准服务保的是基本盘——系统不出大事，出了大事有人救。对于业务相对稳定、规模不大的团队，这是非常扎实的底线保障。"
推介高级成功服务（升级话术）
对比引导
"标准服务是底线保障，高级成功服务是主动经营。最大的区别在于：您有了一个专属技术经理，他是您在金蝶的专属管家。不只是出了问题来处理，更是帮您定期梳理应用现状、提前发现隐患、在月结/年结这些关键时刻为您护航。"
关键时刻场景
"比如每年年末，财务封账、审计同步进来，这是最容易出问题的时候。高级服务里有2个人天的关键时刻专家在线护航，专家全程陪同，出了问题第一时间有人在。这个价值，远不是单次服务费能衡量的。"
定制开发场景
"我知道您的系统有定制开发的部分。标准服务对定制开发问题是不管的——这是一个很大的盲区。高级服务首年会赠送定制开发维护支持，帮您的二开功能做运行异常排查和修复，把这个风险口先堵上。"
IM即时响应场景
"另外，有了高级服务，我们会建立一个专属的服务群，把您这边的关键用户拉进来。工作时间有问题在群里发一下，即时响应——不用再打热线等转接、提工单等回复，体验完全不一样。"

拜访前必备：关键探询问题
  1. 您现在出了问题，主要通过什么渠道联系我们？等待时间能接受吗？ → 挖掘响应速度痛点，引入在线工单和IM服务群价值
  2. 您有没有经历过月结、年结或系统上线时出问题的情况？ → 引出关键时刻支持的场景，场景感最强
  3. 您的系统有没有做过定制开发或接口集成？出问题一般找谁？ → 触发定制开发维护的需求，标准服务的盲区
  4. 您有没有用私有化部署（私有云）？运维这块有专人负责吗？ → 私有云运维是高级服务独有，标准不含
  5. 您现在能清楚了解系统的健康状态吗？数据是否准确、完整？ → 引入系统健康检查和专属服务报告的价值
  6. 您的IT团队对金蝶系统有多熟悉？有没有专门的系统管理员？ → 能力不足 → 专属技术经理是外援；能力强 → 强调效率提升

常见异议处理
🙄 "标准服务就够了"
话术：标准服务保底线，但如果您有定制开发、私有云、或者年结等关键时刻，高级服务是主动风控，事前预防成本远低于事后救火。
💰 "价格太贵"
话术：专属技术经理 + 关键时刻2人天 + 定制开发维护首年赠送，折算单项成本其实相当划算。如果上线时系统出问题影响业务一天，损失是多少？
🤔 "我们IT水平高，不需要"
话术：即便IT能力强，专属技术经理更多是帮您对接金蝶内部资源、协调产品问题推进，这是外部渠道买不到的"内通能力"。
⏰ "已经有标准服务，续费就行"
话术：建议今年先走一个系统健康检查，看看系统现状，再决定是否升级，数据说话。数据会告诉我们答案。
⚠️ 重要提示：合同金额 < 5000 元时，高级成功服务不含「关键时刻支持」人天，推介前需确认合同金额，避免承诺过度。
