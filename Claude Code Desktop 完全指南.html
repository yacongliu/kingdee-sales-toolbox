入门 · 安装与配置

5分钟上手
1 · 下载安装

访问 claude.ai/download 下载桌面版，支持 macOS / Windows。

安装后打开，用 Claude 账号登录（需 Pro/Max/Team 订阅）。

💡

提示：如果你已经在用 CC Switch，安装后可以直接在 CC Switch 中配置模型切换，让桌面版走千问或其他模型。

2 · 连接你的工具

点击左下角头像 → 设置 → 连接器，可连接：

🔗

MCP 服务器

连接飞书、浏览器、数据库等外部工具，让Claude直接操作你的业务系统。

📂

本地文件夹

挂载你的项目目录、销售方案库、客户资料夹，Claude可直接读写文件。

3 · 认识界面
1

主聊天窗口，直接下达指令
2
侧边栏

3

持久化HTML看板，数据实时刷新
02

Projects · 项目管理

给每个客户一个专属空间

Projects 是 Claude Code Desktop 的核心组织单元。每个 Project 有自己的文件目录、上下文记忆（CLAUDE.md）、MCP 连接和技能配置。切换 Project 就是切换整套工作环境。

🎯

核心价值：Project 隔离了不同客户/项目的上下文，防止串项目。这是你之前提到的"每个客户独立窗口"需求的完美解决方案。

4 · 创建Project
1
新建文件夹
在本地创建一个项目目录，比如 ~/Clients/万达电影/
2
在桌面端打开
Claude Code Desktop 中点击"Open Project"，选择该目录
3
编写 CLAUDE.md
写入客户背景、关键联系人、当前进展，以后每次打开自动加载
4
配置 MCP
按需连接飞书、Tavily搜索等工具
5 · CLAUDE.md 模板
✏️

在每个 Project 根目录创建 CLAUDE.md，Claude 每次进入自动读取，作为持久化项目记忆。

# [客户名称] 打单项目 ## 客户信息 - 行业：军工/制造/零售 - 规模：年营收XX亿，员工XX人 - 决策链：CEO（张总）→ CIO（李总）→ 财务（王总） - 内线：生产部 赵经理 ## 项目状态 - 当前阶段：方案汇报完成，待演示 - 关键痛点：项目利润算不准，超支率25% - 竞品动态：用友已接触，出了初步方案 - 下一步：安排针对性产品演示 ## 文档索引 - 客户调研报告：research.md - 定制方案书：proposal_v3.docx - 痛点挖掘速填：quick-fill.md

03

Live Artifacts · 活页面

一次创建，永远新鲜

Live Artifacts 是 Claude Code Desktop 中的持久化、可交互的HTML面板。与普通对话不同，它独立存在，每次打开时自动刷新数据。

📌 持久化

不依赖对话，在"Live Artifacts"标签页永久保存

🔄 实时刷新

打开时自动拉取飞书/日历/搜索等最新数据

📋 版本历史

自动保存每次迭代，可对比差异、还原旧版

7 · 创建方式
1
对话中创建
跟Claude说"帮我创建一个XX看板"或"生成一个持续更新的XX页面"
2
从 Artifacts 标签页
点"New artifact"按钮，直接描述需求
3
现有对话转 Artifact
对话中生成的HTML内容，可以"保存为Artifact"

04

高阶玩法

从能用 → 好用 → 离不开
9 · 并行会话 + 侧边栏

新版桌面支持同时运行多个会话。你可以：

  • 左侧窗口跑客户调研，右侧窗口写方案——互不干扰
  • 主会话做打单推进，Side Chat 快速查询政策/案例——不用切换上下文
  • 跨 Project 拖拽文件引用——调研报告直接拖到方案项目里
10 · 权限模式（Permission Modes）
模式说明适用场景
Ask每步操作都确认关键文件操作、删改
Auto-accept edits允许自动编辑文件写方案、改文档类任务
Plan只规划不执行策略分析、方案框架设计
Auto完全自主（Team/Enterprise）有经验的用户加速工作
⚡

建议：日常用 Plan + Auto-accept 组合——先让Claude出方案你确认，确认后自动执行。

11 · 跨端切换 + 定时任务

📱 从桌面到手机

输入 /desktop 切换到桌面端；点击"Continue on web"推送到云端，手机也能继续聊。

⏰ 定时任务

"每天早上6点查竞品新闻发到Artifact"——Claude自动执行，结果推送到你的Live Artifacts。

12 · 集成终端 + 内嵌预览

桌面端内置终端和编辑器：

  • 编写H5报告 → 直接运行预览，Claude会检查bug并自动修复
  • 写Python调研脚本 → 终端跑完出结果，不用切窗口
  • Diff 视图 → 修改前后对比一目了然，点击"Review code"让Claude审查逻辑

06

优化提示词

向Claude发指令的正确姿势
16 · 三段式模板：背景+任务+格式

一个好 Prompt = 告诉Claude三件事：你是谁→要什么→怎么给

【背景】我是金蝶的销售，正在跟进XX制造集团的ERP项目。 【任务】帮我分析这个客户的痛点，结合行业趋势，给出差异化竞争策略。 【格式】用表格呈现：痛点→根因→金蝶方案→竞品应对→行动建议。
17 · 角色约束法

给Claude一个身份，输出质量明显提升：

🎭

19 · ERP销售专用Prompt模板
场景Prompt
客户调研"我是金蝶天津的销售，正在跟进[企业名]。帮我用Tavily查一下它的工商信息、业务概况、数字化系统现状，出结构化报告。"
痛点分析"针对[企业名+行业]，列出5个最常见的ERP升级痛点，每个痛点包含：场景描述+量化损失+我们的应对方案。"
竞品应对"客户提到用友比我们便宜30%。用TCO对比法写出三个反驳角度，语气专业不贬低，带数据支撑。"
方案框架"针对[企业名]的[痛点]，写一个方案框架：项目背景→解决方案→实施计划→ROI测算→为什么选金蝶。"
谈判话术"客户领导不认可工作量。帮我出两套方案：一套分步走不大幅降价，一套最小方案先见效。每套带具体话术。"
